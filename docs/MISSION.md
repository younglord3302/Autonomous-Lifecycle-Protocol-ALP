# Mission

Create the **universal open protocol** that enables any AI agent to understand, plan, execute, and verify software development autonomously.

## What This Means

1. **Universal:** ALP works with any AI agent, any LLM, and any development environment. No vendor lock-in.
2. **Open:** ALP is an open standard with public specification, schemas, and reference implementations. Anyone can implement it.
3. **Protocol:** ALP is not a tool or a product. It is a format specification — a contract between developers and AI agents.
4. **Understand:** AI agents can parse structured project definitions without LLM inference on unstructured text.
5. **Plan:** AI agents can resolve dependency graphs and determine the correct order of operations.
6. **Execute:** AI agents can follow defined workflows and lifecycle stages to build software.
7. **Verify:** AI agents can check their work against defined quality gates before marking tasks complete.

## Success Criteria

ALP will have achieved its mission when:

- A developer can switch between any two AI coding tools without rewriting their project configuration
- An AI agent can pick up a project mid-stream and understand exactly where the previous agent left off
- A team can define quality standards once and have every agent enforce them automatically
- Multi-agent teams can collaborate on the same project without conflicting modifications
